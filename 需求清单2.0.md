## 一、状态标记体系

| 标记 | 含义 | 说明 |
|------|------|------|
| `✅` | 已打通 | 前后端真实联调通过，可完整使用 |
| `🟡` | 有限制 | API 存在且可用，但有已知限制或降级实现 |
| `🔴` | 待开发 | 依赖外部 API 未接入，后端返回降级响应，前端弹窗提示 |
| `⬜` | 未开工 | 1.0 设计中有，但前后端均未实现 |

**前端降级交互规范**：
- **后端**：返回 HTTP 501 或 JSON `{"error": "功能开发中", "message": "..."}`
- **前端**：弹出 `SnackBar("🚧 功能开发中，下个版本上线")`
